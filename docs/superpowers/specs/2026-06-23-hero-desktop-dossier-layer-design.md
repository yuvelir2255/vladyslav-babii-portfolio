# Hero — Dossier Layer (desktop) — Design Spec

> Дата: 2026-06-23 · Секция: `hero` (`#yard`, экран 1) · Ветка: feature от свежего `main`

## Проблема

На ПК hero ощущается «пустым»: 2-колоночный лейаут (`HeroScreen1.tsx`) с вертикально
отцентрованным контентом оставляет пустыми верх-левую и низ-левую четверти. Левая
колонка «лёгкая» (meta → имя → роль → 2 CTA) против «тяжёлого» фото справа →
дисбаланс масс. Вся «дело-файловая» фактура живёт только в фото; левая половина —
голая типографика. На мобайле колонки складываются в стопку, пустот нет — там всё ок.

Вывод: это проблема **плотности/баланса на широких вьюпортах**, не контента.

## Решение (направление A — «досье-слой»)

Три **статичных** слоя поверх текущего hero. Моушн имени (`HeroNameMotion`/SplitText)
и pin-манифест (`ManifestMotion`) **не трогаем**. Все три слоя — **только desktop
(`lg+`)**: мобайл, которым владелец доволен, остаётся побайтово прежним.

### Слой 1 — кроп-марки по 4 углам экрана 1

Оранжевые L-уголки в каждом из 4 углов `section#yard`. Та же визуальная азбука, что у
`.ev-corner` (forensic-inspect на фото) — консистентность. Задают «дело в рамке»,
убирают пустоту по краям.

- Реализация: 4 абсолютных `<span>` с `border-l/-t` (и зеркально) цвета `--color-orange`.
- Размер ~ 14–18px, отступ от края ~ 16–20px. `pointer-events-none`, `aria-hidden`.
- Видимы только `lg+` (`hidden lg:block`).
- Z-слой: над фоном, под контентом/курсором (например `z-[1]`), кликам не мешают.

### Слой 2 — вертикальный лейбл по левому краю

Текст `CONFIDENTIAL · DO NOT DISTRIBUTE`, `writing-mode: vertical-rl` + `rotate(180deg)`
(читается снизу-вверх), mono-шрифт, цвет `--color-dim`, крупный трекинг.

- Якорит левое поле, закрывает верх-лев дыру.
- Позиция: абсолютно у левого края секции, вертикально по центру.
- Только `lg+`. `aria-hidden` (декор; основной нарратив дублируется в meta/тикере).
- Контент из `hero.ts` → `sideLabel: 'CONFIDENTIAL · DO NOT DISTRIBUTE'`.

### Слой 3 — рап-шит (стат-строка)

Под ролью, отделён сверху тонкой линией (`border-t var(--color-line)`), формат
`LABEL · VALUE`: mono-лейбл (`--color-dim`/`--color-steel`) + значение (`--color-bone`).
Не дублирует тикер, meta-строку и роль.

**Копирайт (характерный тон — голос персонажа, НЕ фактические заявления, поэтому
ничего не выдумываем):**

| Label | Value |
| --- | --- |
| `DISPOSITION` | `UNREPENTANT` |
| `M.O.` | `BUILDS, SHIPS, ESCAPES` |
| `THREAT LEVEL` | `HIGH` |

- Контент из `hero.ts` → `rapSheet: readonly { label: string; value: string }[]`.
- Только `lg+`. Размещается между `role` и блоком CTA в левой колонке.
- Адаптив узкого desktop: полный набор (3 поля) на `xl+`; на `lg` — 3-е поле
  (`THREAT LEVEL · HIGH`) скрыть (`hidden xl:flex` на 3-м), чтобы не поджать фото на
  ~1024–1100px. Альтернатива при тесте — перенос в 2 строки; решается в браузере.

## Контент-модель (`src/content/hero.ts`)

Добавить два поля (типизировать `as const`):

```ts
sideLabel: 'CONFIDENTIAL · DO NOT DISTRIBUTE',
rapSheet: [
  { label: 'Disposition', value: 'Unrepentant' },
  { label: 'M.O.', value: 'Builds, ships, escapes' },
  { label: 'Threat level', value: 'High' },
] as const,
```

Регистр в UI — через `uppercase` (как у остальных лейблов hero), в данных хранить
человекочитаемо.

## Разметка (`src/components/hero/HeroScreen1.tsx`)

- Кроп-марки: 4 `<span>` прямыми детьми `<section id="yard">` (внутри, абсолютные),
  `hidden lg:block`.
- Вертикальный лейбл: `<span>` абсолютный у левого края секции, `hidden lg:block`.
- Рап-шит: блок между `<p>{role}</p>` и `<div>…CTA…</div>` внутри `HeroNameMotion`,
  обёртка `hidden lg:flex`, элементы `.map(rapSheet)`.

NB: чтобы кроп-марки/лейбл не клипались, секция уже `relative overflow`-нейтральна
(глобально `html{overflow-x:clip}` страхует H-скролл). Уголки держать **внутри**
вьюпорта (отступ от края), как `.ev-corner`.

## CSS / классы

Предпочесть Tailwind arbitrary inline (как в текущем hero). Если потребуется новое
правило в `globals.css` (напр. общий `.hero-corner`) — помнить **NB dev-кэш**: новое
CSS-правило/класс Turbopack в dev не подхватывает без `Remove-Item .next` + рестарт.
По возможности обойтись инлайном, чтобы не плодить правила.

Цвета — только токены (`--color-orange`/`--color-dim`/`--color-steel`/`--color-bone`/
`--color-line`). Без сырых hex.

## A11y

- Кроп-марки и вертикальный лейбл — `aria-hidden="true"` (чистый декор).
- Рап-шит — осмысленный текст, не прячем от screen-reader (но `lg`-only через CSS
  `display`, не `aria-hidden`); порядок чтения логичный (после роли, до CTA).
- Контраст: `--color-dim` (4.62:1) и выше — проходят AA. Значения на `--color-bone`.
- `:focus-visible`, навигация с клавиатуры — без изменений (новые слои некликабельны).
- `prefers-reduced-motion` — неактуально (слои статичные).

## Перформанс

- Всё статично, без новых анимаций/RAF/will-change.
- Никаких новых изображений/шрифтов (используем уже подключённые mono/label).
- LCP (мугшот) не затрагивается.

## Адаптив (контроль в браузере, preview-инструмент)

- 375 / 768: новые слои **отсутствуют** (`lg`-gated) → мобайл идентичен текущему.
- 1024–1100 (`lg`): 2 поля рап-шита, фото не поджато, без H-скролла.
- 1280 / 1440 (`xl`): полный набор, кроп-марки в углах, вертикальный лейбл слева.

## Тесты (Vitest, jsdom)

- Контент: `hero.ts` экспортит `sideLabel` (string) и `rapSheet` (массив длиной 3,
  у каждого `label`+`value`).
- Рендер `HeroScreen1`: присутствуют значения рап-шита (по тексту/`data`-атрибуту) и
  `sideLabel`; кроп-марки помечены `aria-hidden`. Моушн в юнит-тестах не гоняем.
- `npm test` остаётся зелёным; добавленные тесты считаем поверх текущих 75.

## «Готово» = доказано

- Чистый `npm run build` + `npm test` зелёный + `npm run typecheck`.
- Проверка в preview: 375/768 (мобайл не изменился) и 1280/1440 (новые слои на месте,
  без H-скролла, фото не поджато). Скрин-доказательство ПК + мобайл.

## Вне scope (YAGNI)

- Направления B (перекомпоновка сетки) и C (фоновый watermark/линейка) — не делаем.
- Числовые метрики — отклонены владельцем (нечисловой флавор-копирайт).
- Любые правки моушна, фото, других секций.

## Откат

Изолировано в `hero.ts` + `HeroScreen1.tsx`. Откат — revert коммита фичи. Прод не
трогаем до явного «ок» владельца (feature-ветка → preview-деплой → ревью).
