## 4. Спецификация по компонентам

### A. Фон — FBM-дым на OGL (`src/components/field/Field.tsx` — переписать)

Заменяем текущую сцену (дюны + звёздное небо на точках) на **полноэкранный квад с фрагментным дымовым шейдером** — это дешевле particle-системы Luca, но даёт тот же эффект «клубящегося дыма, который курсор расступает».

- **Шум:** FBM (3–4 октавы) поверх value/simplex-шума; **domain warp** для клубящейся структуры. Дрейф по `uTime`.
- **Цвет:** градации серого на near-black фоне (`--color-bg`), мягкое внутреннее свечение, тонкий вертикальный вигнет. Без цветных тинтов (монохром).
- **Курсор раздвигает дым** (как у Luca, униформы из его шейдера):
  - `uMouse` (позиция, сглажена lerp на стороне JS),
  - `uMouseRadius` — радиус влияния,
  - `uMouseStrength` — сила domain-warp вокруг курсора (искривление),
  - `uMouseHole` — локальное прореживание плотности («дырка» в дыму под курсором),
  - опц. лёгкое свечение по краю «дырки».
- **Скролл:** медленный сдвиг/затухание дыма по прогрессу скролла (сохранить идею `uScroll` из текущего Field). Velocity скролла может слегка усиливать клуб (тонко).
- **Переиспользуем** существующую обвязку из текущего `Field.tsx`: рендер-луп rAF, ресайз, проводку курсора и скролла, OGL `Renderer/Program/Mesh`.
- **Убираем:** геометрию дюн, звёздное небо, мерцание, падающие звёзды, `uEnergy/uTint` дюн (если не переносятся в дым осмысленно).
- **Перф:** `dpr` cap (≤2 десктоп, ≤1.5 мобайл); на `<768` меньше октав FBM; пауза рендера, когда канвас вне вьюпорта (IntersectionObserver) и при `visibilitychange`; антиалиас не нужен (полноэкранный квад).
- **Фолбэк:** без WebGL/JS — статичный near-black фон (как сейчас). Grain (`ui/Grain.tsx`) поверх остаётся.

**Acceptance:** дым клубится; курсор видимо раздвигает/искривляет дым и оставляет прореженный след; монохром; 60fps десктоп, без явных просадок на мобайле 375; 0 ошибок в консоли; нет горизонтального скролла.

### B. Типографика — IBM Plex Mono везде

- **`src/app/[locale]/layout.tsx`:** заменить `Archivo` + `Space_Mono` на `IBM_Plex_Mono` (`next/font/google`, OFL/бесплатный). Веса: 300/400/500/600 (+ italic при необходимости). Одна переменная (напр. `--font-plex`), `display: 'swap'`, `subsets: ['latin']` (+ `latin-ext`/`cyrillic` если нужно для UK — **проверить покрытие кириллицы у Plex Mono**; если нет — fallback на системный моно для UK-кириллицы).
- **`src/app/globals.css @theme`:** `--font-sans` и `--font-mono` оба → `var(--font-plex), ui-monospace, monospace`. (Дисплей у Luca тоже моно — отдельного sans не держим.)
- **Шкала:** флюидная (`clamp`) для hero/заголовков, тесный `line-height` (≈1.0–1.2) на крупном, 1.5 на body. Моно-цифры/буквы шире — **проверить переносы** во всех секциях и на 375 (имя в Hero, BUILD/CRAFT, лиды).
- **Микро-лейблы (эйбрау секций):** `text-transform: uppercase` + `letter-spacing ≈ .2–.25rem` — трекинг-капс разметка, как у Luca.
- Удалить из проекта `Archivo`/`Space_Mono` (импорты, className `html`).

**Acceptance:** на сайте один шрифт — IBM Plex Mono; переносы/ритм выверены 375/768/1280; UK-кириллица читается (или корректный fallback); build чистый.

### C. Курсор ⇄ дым (`src/components/ui/Cursor.tsx` + Field)

- Комета-курсор сохраняется (хвост, кольцо на интерактиве через делегирование `pointerover/out`, магнитные CTA через `ui/Magnetic`).
- Курсор **кормит `uMouse`** дымового шейдера (общий источник позиции мыши, сглаженный lerp). Над интерактивом — кольцо курсора + при желании усиление `uMouseStrength`/`uMouseHole` (дым активнее расступается у кнопок).
- Связь курсор↔Field сделать через лёгкий общий стор/ref или существующий механизм проводки мыши в Field (без новых зависимостей).

**Acceptance:** движение мыши одновременно двигает комету и раздвигает дым; над ссылками/кнопками реакция усилена; 0 ошибок.

### E. Моушн-язык (выравнивание почерка)

**Acceptance:** скролл и ревилы ощущаются единым почерком; наши фирменные моменты живы; overflow нет на 375/768/1280; Playwright-моушн подтверждает.

## 5. Риски и заметки

- **Кириллица в IBM Plex Mono:** проверить покрытие для UK; при пробелах — fallback-моно для кириллицы (не ломать UK).
- **Дым vs читаемость:** текст не должен тонуть в дыму — сохранить/настроить `.legible`-гало; дым держать приглушённым под контентом (плотность/яркость ниже у краёв контентных блоков или общий low-contrast).
- **Перф мобайла:** дым на фрагментном шейдере дешевле particle-системы, но FBM с domain-warp тяжёлый — обязательно cap dpr + меньше октав + пауза вне вьюпорта.
- **Чистый монохром** убирает цветовую айдентику — узнаваемость держим курсором + контентом + hero-частицами (осознанный выбор владельца).
