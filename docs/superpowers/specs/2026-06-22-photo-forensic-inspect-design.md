# Photo «Forensic Inspect» hover — спек

> Дата: 2026-06-22 · Ветка: `feat/photo-forensic-hover` · Концепт-родитель: «Case File».

## Цель

Единая ховер-сигнатура **«улику достали на осмотр»** на ВСЕ фото сайта, в «жирной» (подраматичнее) версии. Фото — это «улики дела»; при наведении курсора лист дела как будто вытаскивают поближе к глазам и сканируют.

## Где применяется

| Место | Фото | Файл |
| --- | --- | --- |
| Hero (`#yard`) | `mugshot.webp` (booking-фото) | `components/hero/BookingPhoto.tsx` |
| About (`#about`) | `inmate-build.webp` (цвет, главный) | `components/about/About.tsx` |
| About (`#about`) | `inmate-detail.webp` (ч/б, «Profile») | `components/about/About.tsx` |
| Work (`#work`) | скрины Dream Gold в рамке-телефоне | `components/work/ExhibitDevice.tsx` |

## Сигнатура (5 слоёв, срабатывают разом при `:hover`)

1. **Zoom** — изображение `scale 1 → 1.08` внутри рамки `overflow:hidden`; ease `cubic-bezier(.2,.7,.2,1)`, ~500 мс.
2. **Свечение рамки** — бордюр `var(--color-line)/steel → var(--color-orange)`; наружу мягкое оранжевое `box-shadow`-зарево (заметно ярче статичного состояния).
3. **Кроп-марки** — 4 угловых уголка-визира «защёлкиваются» внутрь (translate) + ярчают (эффект фокусировки камеры).
4. **Скан-линия** — ОДИН проход тонкой оранжевой полосы/градиента сверху→вниз по фото при наведении (CSS-`@keyframes`, проигрывается на каждый `:hover`). Главный «больдовый» акцент.
5. **Лифт тега** — evidence-тег (`EXHIBIT A` / `№VB-19` / `EVID-01`) чуть приподнимается (`translateY`) + ярчает.

## Механика и ограничения

- **Чистый CSS**, без GSAP/JS — как уже сделаны `RollText` и штампы (устоявшийся приём проекта). Анимируем только `transform` / `opacity` / `box-shadow` / `border-color` (перф, без layout-shift).
- **Только desktop с мышью:** всё внутри `@media (hover:hover) and (pointer:fine)`. На тач — фото в состоянии покоя (вход уже даёт scroll-reveal; tap на Work занят навигацией карусели).
- **`prefers-reduced-motion` НЕ уважаем** (политика проекта — движение всегда вкл).
- **Переиспользуемость:** набор утилит в `src/app/globals.css` — `.ev-frame` (обёртка `overflow:hidden` + рамка/зарево), кроп-марки на `::before/::after` (или вложенный span), `.ev-scan` (скан-линия). Минимальная разметка-обёртка вокруг каждого фото. Один язык — четыре применения.
- **Сохранить всё существующее:** `alt`/`aria-hidden`, `priority`, `sizes`, `data-*`-атрибуты, теги/штампы, размеры `next/image`. Ничего не удаляем — тесты ищут по DOM/`data`.

## Применение по местам (нюансы)

- **Hero `mugshot`** — полная сигнатура; лифтит `EXHIBIT A`; штамп `At Large` и `Placard` не трогаем.
- **About `inmate-build`** (цвет) — полная; на ховере убрать текущий `grayscale-[0.15]` → «доводка в резкость» (в духе A, НЕ колоризация).
- **About `inmate-detail`** (ч/б, `aria-hidden`, мелкий) — та же сигнатура, масштаб чуть мягче; остаётся ч/б (колоризацию не делаем).
- **Work / телефон** — zoom применить к ВНУТРЕННЕМУ слою экрана (обёртка вокруг `shots.map`), чтобы кроссфейд кадров (`opacity`) не ломался; рамка-зарево + ярчает существующий пунктир «пакета» (`data-bag-draw`); реагируют тег `EVID-01` и скан-линия по экрану; стрелки/точки-индикаторы остаются кликабельны (ниже устройства, отдельный блок). Кроп-марки на прямоугольный телефон не вешаем — акцент на пунктире-пакете.

## Проверка («готово» = доказано)

- Чистый `npm run build`.
- `npm test` зелёный (мотон в юнитах не гоняем; проверяем, что разметка не сломала существующие DOM-тесты).
- Живой прогон в браузере: 1280 (ховер по всем 4 фото — все 5 слоёв), 375 (покой, без горизонтального скролла), 768. Скрин/гиф владельцу.
- Деплой в `main` — только по явному «ок» (обычный конвейер ветки).
