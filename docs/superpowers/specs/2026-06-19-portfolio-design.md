## 1. Цель и аудитория

- **Что это:** персональное портфолио-сайт, который **сам по себе является доказательством навыков** (3D/WebGL, анимации, моушн) — «show, don't tell».
- **Аудитория:** работодатели и потенциальные клиенты, которым Влад отправляет ссылку.
- **Главное действие (конверсия):** написать через **форму контакта → Telegram** (заявка на заказ, напр. Telegram-приложение).
- **Критерии успеха:**
  1. Уровень моушна/качества — сопоставим с aboutluca (awwwards-класс).
  2. Понятно за 1 экран, кто это и что умеет; за 1 скролл — доказательство (кейс Dream Gold).
  3. Двуязычность EN/UK, быстрый отклик, без визуальных дефектов на 375/768/1280.

## 3. Стек и архитектура

Свежий проект на стеке Влада (совпадает с его dream-gold/vlad-portfolio):

- **Next.js (App Router) + React + TypeScript (strict) + Tailwind CSS.**
- **Анимации:** GSAP (ScrollTrigger, SplitText) + Lenis (плавный скролл) + Framer Motion (точечно для UI).
- **3D/WebGL:** поле частиц — **OGL** (лёгкий, Влад уже использовал) либо Three.js. По умолчанию OGL.
- **i18n:** next-intl (EN/UK) + тест паритета ключей.
- **Контакт-форма:** Vercel serverless route → Telegram (`TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID`), как в dream-gold.
- **SEO/OG:** localized metadata, `next/og`, sitemap/robots, hreflang.
- **Тулинг:** Vitest, ESLint, Prettier, Husky + lint-staged.
- **Деплой:** Vercel (новый проект). Текущий статичный about-me-website живёт, пока новый не готов; затем переключаем домен.
- **Репозиторий:** новый (рабочее имя `portfolio`). Решим на этапе плана.

## 6. Дизайн-система

- **Палитра:** база — почти-чёрный (#0a0b0d / #101114), текст #f4f4f2, приглушённые серые. Акценты (только карточки/детали): синий, голубой, фиолетовый, изумрудный, янтарный — приглушённые, «стеклянное свечение».
- **Типографика:** display — крупный гротеск (Archivo Expanded / похожий), лейблы и техно-надписи — моно (Space Mono), body — гротеск. (Финал — на этапе дизайн-конвейера.)
- **Сетка/отступы:** mobile-first; брейкпоинты 375 / 768 / 1280; крупный «воздух», как в кинопремиум.
- **Курсор:** кастомный (перекрестие/точка), магнитные состояния на интерактиве.
- **Текстуры:** лёгкий film-grain для кинематографичности; виньетка по краям.

## 7. Интерактив и моушн-принципы

- **Всё «дышит»:** ничего не «прыгает», все переходы плавные (ease, scrub на скролле).
- **Скролл = камера:** прокрутка двигает 3D-сцену, растит дюны, проявляет слова (GSAP ScrollTrigger + Lenis).
- **Текст:** SplitText, проявление/исчезновение по словам.
- **Карточки:** последовательное появление с приближением; hover-динамика.
- **Hover везде:** карточки и ключевой текст реагируют микро-анимацией.
- **Звук:** опционально, Phase 2 (тумблер mute, выкл по умолчанию) — чтобы не раздувать скоуп и не искать ассеты.
- **prefers-reduced-motion:** облегчённая версия (статичное поле/без падающих звёзд, мгновенные reveal).

## 8. Контент-модель и форма

- **Проекты — типизированная контент-коллекция** (типизированный TS + MDX для тел кейсов; **решено** — без внешней CMS, макс. контроль/качество), чтобы добавлять новые проекты без переписывания секций. Поля: slug, title, role, summary, tags, media[], links[], status (`live` / `coming-soon`), locale-копирайт.
- **Форма контакта:** POST `/api/contact` → валидация → Telegram Bot API (`sendMessage` в `TELEGRAM_CHAT_ID`). Антиспам (honeypot/rate-limit). Секреты только на сервере.

## 9. i18n (EN/UK)

- next-intl, локали `en` (по умолчанию) и `uk`, переключатель в шапке.
- Весь новый текст — сразу в обе локали; Vitest-тест паритета ключей валит сборку при пропуске.

## 10. Перформанс и доступность

- WebGL: адаптивное число частиц (меньше на мобиле/слабом GPU), пауза рендера вне вьюпорта, дегрейд при `prefers-reduced-motion`.
- LCP бережём (прелоадер не блокирует бесконечно; таймаут-фолбэк).
- a11y: контраст ≥ 4.5:1, видимый `:focus-visible`, `aria-label` у иконок-кнопок, навигация с клавиатуры, тач-цели ≥ 44px, форма с лейблами и ошибками.

## 11. SEO

- Локализованный `generateMetadata`, canonical + hreflang (en/uk/x-default), OG-картинки (`next/og`, кириллица-safe), `sitemap.ts`, `robots.ts`, один `<h1>`, семантический HTML.
