# Дизайн-спека: Services — плавный переход count 05 → VERDICT («Collapse»)

> Дата: 2026-06-21. Статус: согласовано владельцем в брейншторме.
> Точечное **улучшение моушна** существующей секции `#services`. Развивает
> `2026-06-21-services-charges-design.md` (та же дизайн-система «Case File»: бетон + сталь +
> prison-orange). **Скоуп — ТОЛЬКО финальный сегмент перехода 05 → вердикт.** Пункты 01→05,
> их `revealCount`, рейл, snap-дека, контент — НЕ меняем.

## 1. Проблема и цель

Сейчас последний шаг (count 05 → экран `GUILTY ON ALL FIVE COUNTS`) — **дискретный кроссфейд**:
разовый `show(verdict)` гасит карточку/рейл/хедер и проявляет вердикт за `0.25s`, плюс прыжок
самого `snap`. Владелец: переход ощущается **резким**.

**Цель:** сделать переход 05 → вердикт **супер-плавным, привязанным к скроллу** (тянется за
скроллом, отматывается вверх), в тоне **«плавно-кинематографично»** (спокойный заход, без
резких ударов). Концепт — **🅰 Collapse**: композиция обвинения схлопывается к центру и
собирается в приговор.

## 2. Механика (ключевое изменение)

Переводим **только финальный сегмент** (между snap-точкой count 05 и концом пина) из дискретного
кроссфейда в **непрерывный scrub**:

- Пункты `01 → 05` остаются дискретной snap-декой с `revealCount(i)` — без изменений.
- Когда `self.progress` входит в **последний сегмент** (индекс 4 → 5), вместо бинарного
  `show()` считаем **sub-прогресс сегмента** `p ∈ [0..1]` и применяем коллапс-трансформы
  **континуально** в `onUpdate`. Природа `scrub` + Lenis даёт плавность; скролл вверх отматывает.
- Выход из сегмента вверх (обратно к 05) — корректный сброс трансформов (возвращаемся к обычной
  деке). Идемпотентно: каждый кадр пересчитываем из `p`, без накопления состояния.

## 3. Хореография (по sub-прогрессу `p` сегмента)

**Уходит — «обвинение» (`cg` = хедер + активная карточка 05 + рейл):**
1. `scale 1 → ~0.84`, `opacity 1 → 0` (к ~`p≈0.6`), лёгкий `blur 0 → 6px`,
   `transform-origin: 50% 50%` — схождение к центру.
2. Рейл подъезжает влево (`translateX ~ -34px`), плашка `Guilty ×5` дрейфует к центру и тает.

**Приходит — вердикт (с нахлёстом, проявление ~`p 0.4 → 1`):**
3. Заголовок `GUILTY ON ALL FIVE COUNTS` — **per-line маска** снизу
   (`SplitText {type:'lines', mask:'lines'}`, `yPercent 100 → 0`, лёгкий stagger по строкам).
4. **Glow-bloom** за заголовком: отдельный div с радиальным оранжевым градиентом,
   `opacity 0 → ~0.5` + лёгкий `scale 0.6 → 1`. **Чистая мягкая фиксация без пульсации**
   (решение владельца; пульс — возможное улучшение позже).
5. Sentence (`Sentenced to keep…`) проявляется последней.

**Разовый акцент (не скрабится):**
6. Айбров `THE COURT FINDS THE DEFENDANT` — **одноразовый `ScrambleText`** при входе в сегмент
   (`onEnter`-стиль). Скрабить scramble нельзя — консистентно с остальными акцентами сайта
   (грифы/штампы). При отмотке вверх и повторном входе — допускается повторный проигрыш.

Интенсивность: **кинематографичная** — без screen-shake и без хард-слэма заголовка (в отличие от
штампов `GUILTY` на counts). Заголовок именно «собирается», а не «бьётся».

## 4. Desktop / mobile (gsap.matchMedia — уже есть)

- **Desktop (`min-width:1024px`):** полный эффект — `scale + opacity + blur(6px) + glow + маска`.
- **Mobile/планшет (`max-width:1023px`):** легче и короче — `scale + opacity + glow + маска`,
  **без `blur`** (GPU-дорого на тач-GPU; перф > эффект). Scramble айброва оставляем.
- Длину пиннинга/тайминг последнего сегмента подстроим, чтобы коллапсу хватало «воздуха»
  (Services сейчас `pct`: desktop 72 → `+=432%`, mobile 52 → `+=312%`). Возможно — чуть больше
  доли скролла именно на финальный сегмент.

## 5. Технологии

- **GSAP ScrollTrigger** — тот же pin+snap; финальный сегмент читаем из `onUpdate`/`self.progress`.
- **SplitText** (`mask:'lines'`) — per-line раскрытие заголовка вердикта.
- **ScrambleText** — разовый scramble айброва.
- **gsap.matchMedia** — desktop (с blur) / mobile (без blur).
- **Lenis** (глобально) — источник плавности scrub.
- **Без новых зависимостей.** Все плагины уже в `src/lib/gsap.ts`. Glow — один div + CSS
  radial-gradient (можно через токен `--color-orange`).

## 7. a11y / правила проекта

- `prefers-reduced-motion` намеренно НЕ уважаем (решение владельца). Glow и заголовок —
  декоративный слой; `aria-hidden` где уместно.
- **Фолбэк-видимость:** при несработавшем ScrollTrigger (SSR/headless/тесты) вердикт и counts
  остаются в DOM и читаемы (паттерн секций). Текущие тесты ищут по DOM/`data`-атрибутам —
  не должны падать.
- Контраст токенов — уже AA; видимый `:focus-visible` не затрагиваем.
- Брейкпоинты 375 / 768 / 1280, без горизонтального скролла.

## 8. Проверка («готово»)

- Чистый `npm run build` + `npm run typecheck`.
- `npm test` зелёный (моушн в юнитах не гоняем — правило проекта).
- Браузер (preview): прокрутка 05 → вердикт **и обратно вверх** на 1280 / 768 / 375 —
  плавно, без рывка, glow мягко всплывает, заголовок собирается маской, scramble айброва
  один раз. Цель 60fps; следим, что blur не роняет fps на desktop.
- Финальную плавность на реальном тач-устройстве проверяет владелец.

## 9. Открытые вопросы (решим на сборке)

- **Точная доля скролла** под финальный сегмент vs остальные snap-шаги — подобрать «на ощущение».
- **Пик glow и его радиус** — финализировать в браузере (тонко, не «неон»).
- **Поведение scramble айброва при быстрой отмотке** вверх-вниз — дебаунс/гард, чтобы не «мигал».
